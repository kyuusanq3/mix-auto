Offline regional maps — implemented (MapLibre OfflineManager)

Architecture
- POI/search: unchanged — ph_places.db.gz from mix-auto-overture-maps via Map Data → Search & places row.
- Basemap: vector-only regional downloads via MapLibre OfflineManager (OpenFreeMap tiles/glyphs/sprites cached on device).
- Style: display = asset `mix-auto-driving.json`; offline pack = minimal `mix-auto-offline-pack.json` (vector only, no glyphs) copied to file:// for OfflineManager.
- Catalog: app/src/main/assets/map/offline_regions.json (PH pilot: overview, Negros Occidental, Western Visayas).
- Code: OfflineMapRepository, MapDataViewModel offline actions, expandable Map Data UI in MapDataOverlay.kt.

UI (Map Settings → Map Data)
- Expandable country card (Philippines): POI pack row + per-region offline map rows.
- Near-you suggestion from GPS pins matching region bbox.
- Storage summary: Places MB + Maps MB.
- Vector-only gate when raster mode selected.
- One download at a time (POI or map region).

HUD
- mapConnectivityLabel in MapUiState — "Offline map cached" when no network + complete region covers GPS (CarMapViewContainer top-start when not navigating).

Future (not implemented)

- Custom bbox picker on map.
- Nav corridor prefetch snackbar.

Don't

- OfflineManager on raster mode.
