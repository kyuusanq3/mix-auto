Mix Auto — Automotive Vector Style (Wider Roads at Nav Zoom)

Problem: At driving zoom (~17–18) roads on the vector map read thin compared to Waze/Google Maps. The default OpenFreeMap Liberty style uses general-purpose `line-width` stops on transportation layers. Raising the zoom slider above ~22 only upscales tiles (blurry, not sharper). Raster OSM mode cannot widen roads — vector tiles only.

Goal: Make roads feel consistently bigger at nav/free-drive zoom without overscaling past tile detail (~22 max useful zoom).

Recommended approach — bundled custom style JSON

1. Fork OpenFreeMap Liberty (https://tiles.openfreemap.org/styles/liberty) or Dark (https://tiles.openfreemap.org/styles/dark — better OLED match for dashboard UI).
   Source repo: https://github.com/hyperknot/openfreemap-styles

2. Boost `line-width` interpolate stops at zoom ≥14 on road layers:
   - Target layers: `road_*`, `tunnel_*`, `bridge_*` (exclude `road_one_way_*`, `road_area_*`, path/pedestrian/rail)
   - ~1.5× fill width; ~1.7× for minor/residential + matching casing layers
   - Optional: bump `highway-name-minor` / `highway-name-major` text-size +1–2 px at z16+

3. Save as `app/src/main/assets/map/mix-auto-driving.json` (or dark variant).

4. Keep the same free tile/sprite/glyph URLs (no API key; still needs INTERNET):
   - `"openmaptiles": { "type": "vector", "url": "https://tiles.openfreemap.org/planet" }`
   - sprite + glyphs URLs unchanged from Liberty

5. Wire in `MapLibreEngineImpl.applyMapStyle()`:
   - Replace remote `VECTOR_STYLE_URL` with `Style.Builder().fromUri("asset://map/mix-auto-driving.json")` when automotive style enabled
   - Or always use bundled style for vector mode

Must preserve layer IDs (existing code depends on them):
- `poi_r1`, `poi_r7`, `poi_r20`, `poi_transit` — POI tap / vector POI query
- `road_motorway` — route/overlay anchor fallback in `resolveMapOverlayAnchorLayerId()`
- `building-3d` / FillExtrusionLayer — 3D buildings toggle (`apply3dBuildingVisibility`)
- `osm` — raster mode only (unchanged)

Alternative MVP — runtime style patch

- Add `applyAutomotiveRoadBoost(style)` called from the `setStyle` callback (mirror `apply3dBuildingVisibility()`).
- Loop `style.layers`; for each `LineLayer` with id prefix `road_` / `tunnel_` / `bridge_`, set boosted `PropertyFactory.lineWidth(Expression.interpolate(...))`.
- Harder to maintain than a JSON fork (~30 layers with different stop tables).

Optional UX

- Settings toggle: "Bold roads" in Map Settings (`LauncherPreferences` + `MapSettingsPanelContent`)
- Default ON for vector tiles; no effect when raster mode selected

Testing

- Vector tiles ON, driving zoom 17.5, tilt 50° free drive / 58° nav
- Compare minor street, secondary, motorway width on Eonon head unit
- Verify POI tap, 3D buildings toggle, traffic overlay, route line layering still work

Reference: `MapLibreEngineImpl.kt` — `applyMapStyle()`, `VECTOR_STYLE_URL`, `apply3dBuildingVisibility()`
